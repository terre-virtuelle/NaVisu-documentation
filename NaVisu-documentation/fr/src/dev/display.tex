\section{Architecture}
\subsection{Principes}
Un {\tt Display} est un composant, au sens de C$^3$, il peut ainsi bénéficier des services du 
composant {\tt GuiAgent} par exemple. L'application n'a aucun lien codé avec lui. Le design du nouveau compo\-sant 
est complètement séparé du code.
Chaque élément variable possède un id. Le contrôleur, instancié par le {\tt Display} lors de l'initialisation
charge le fichier {\tt .fxml}. On retrouve dans le code du contrôleur les id de la partie graphique.
Préconisations : le composant graphique principal est un {\tt Group}, son id est : {\tt view}, chaque {\tt Display}
possède un bouton de fermeture, dont l'id est {\tt quit}. Le contrôleur hérite de la classe {\tt Widget2D} qui offre 
les services de l'interactivité.
 L'attribut {\tt KEY\_NAME}, ici : {\tt "InstrumentTemplate"} permet au Dock de le rechercher et de l'afficher, lorsque l'item
 associé est sélectionné. Comme ci dessous dans la classe {\tt DockManagerImpl} du module {\tt navisu-app}, lors de
 la création du Dock :
 {\small 
\begin{verbatim}
instrumentsRadialMenu = RadialMenuBuilder.create()
.centralImage("instrumentsradialmenu150.png")
.createNode(0, "navigation.png",1,"ais.png",1,"template.png",(e)->open("InstrumentTemplate"))
.build();
\end{verbatim}
}
Le nouveau {\tt Display} devra s'enregistrer auprès du {\tt IntrumentDriverManagerServices}, dans la classe {\tt AppMain} du module {\tt navisu-launcher}.
{\small
\begin{verbatim}
 // deploy components
 LOGGER.info("\n"
 + componentManager.startApplication(DpAgentImpl.class,
 .....
  InstrumentTemplateImpl.class
  )
  );
  ......
  InstrumentTemplateServices instrumentTemplateServices 
                   = componentManager.getComponentService(InstrumentTemplateServices.class); 
  ......
  instrumentDriverManagerServices.registerNewDriver(instrumentTemplateServices.getDriver());
\end{verbatim}
}

\section{Créer un nouveau {\tt Display} : {\tt Compass}}
Reprendre le code du {\tt InstrumentTemplate}, écrire les interfaces {\tt Compass} et {\tt CompassServices}, implémenter
les classes {\tt CompassImpl} et {\tt CompassControler}.Créer le fichier graphique {\tt compass.fxml}. Identifier par un id chaque variable.
Reprendre ces variables en mode public dans le contrôleur. Eventuellement ajouter des services, par défaut
un {\tt Display} offre le service {\tt on()}. Implémenter les contrôles.
